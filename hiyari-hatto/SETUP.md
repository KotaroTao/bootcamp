# ヒヤリハット報告アプリ - セットアップ手順

## 概要

このアプリは **Google スプレッドシートをサーバー代わり** に使い、
複数のスマホ・PCから報告データを共有できます。

```
スマホA ──┐                  ┌── Google スプレッドシート
スマホB ──┼── Apps Script ──┤   （データベース代わり）
   PC  ──┘    （API）       └── 自動で行が追加される
```

---

## Step 1：Google スプレッドシートを作成

1. [Google スプレッドシート](https://sheets.google.com) を開く
2. 「空白のスプレッドシート」を新規作成
3. 名前を「ヒヤリハット報告」に変更

---

## Step 2：Apps Script にコードを貼り付け

1. スプレッドシートのメニューから **「拡張機能」→「Apps Script」** を開く
2. 既存のコードをすべて削除
3. `gas/Code.gs` の内容をすべてコピーして貼り付け
4. 「保存」ボタン（💾）をクリック

---

## Step 3：シートを初期化

1. Apps Script エディタの上部で、関数選択を **`initSheet`** に変更
2. 「実行」ボタン（▶️）をクリック
3. 初回は権限の承認が必要です：
   - 「権限を確認」をクリック
   - 自分のGoogleアカウントを選択
   - 「詳細」→「〇〇（安全ではないページ）に移動」をクリック
   - 「許可」をクリック
4. スプレッドシートに「報告データ」シートが作成され、ヘッダー行が追加されます

---

## Step 4：ウェブアプリとしてデプロイ

1. Apps Script エディタの右上 **「デプロイ」→「新しいデプロイ」**
2. 左上の歯車アイコン →「ウェブアプリ」を選択
3. 以下のように設定：
   - **説明**：ヒヤリハット報告API
   - **実行するユーザー**：自分
   - **アクセスできるユーザー**：**全員**
4. 「デプロイ」をクリック
5. 表示された **ウェブアプリのURL** をコピー

> ⚠️ URLは `https://script.google.com/macros/s/XXXXX/exec` の形式です

---

## Step 5：アプリに接続

1. `index.html` をブラウザで開く
2. 「設定」タブを開く
3. コピーしたURLを貼り付け
4. 「接続」ボタンをクリック
5. 「接続に成功しました！」と表示されればOK

---

## 複数端末での使い方

接続が完了したら、**同じ `index.html` ファイルを各端末で開く** だけです。

### 共有方法（簡単な順）

| 方法 | 手順 |
|------|------|
| **GitHub Pages** | リポジトリを GitHub Pages で公開し、URLを共有 |
| **USBメモリ** | HTMLファイルを各PCにコピー |
| **社内サーバー** | HTMLファイルを置いて、LANからアクセス |

各端末で初回に「設定」タブからGAS URLを入力すれば、
全端末のデータがスプレッドシートに集約されます。

---

## コードを更新した場合

Apps Script のコードを修正したら、再デプロイが必要です：

1. **「デプロイ」→「デプロイを管理」**
2. 右上の鉛筆アイコン（編集）をクリック
3. バージョンを **「新バージョン」** に変更
4. 「デプロイ」をクリック

> URLは変わりません。アプリ側の変更は不要です。

---

## トラブルシューティング

| 症状 | 原因・対処法 |
|------|-------------|
| 「接続に失敗しました」 | URLが正しいか確認。デプロイ時「全員」にしたか確認 |
| データが表示されない | `initSheet` を実行したか確認。シート名が「報告データ」か確認 |
| 権限エラー | Apps Script の権限を再承認 |
| 送信が遅い | GAS の初回起動（コールドスタート）は数秒かかることがあります |
